# <a name="migrations-overview"></a><span data-ttu-id="2b078-103">迁移概述</span><span class="sxs-lookup"><span data-stu-id="2b078-103">Migrations Overview</span></span>

<span data-ttu-id="2b078-104">在实际项目中，数据模型随着功能的实现而变化：添加和删除新的实体或属性，并且需要相应地更改数据库架构，使其与应用程序保持同步。</span><span class="sxs-lookup"><span data-stu-id="2b078-104">In real world projects, data models change as features get implemented: new entities or properties are added and removed, and database schemas needs to be changed accordingly to be kept in sync with the application.</span></span> <span data-ttu-id="2b078-105">EF Core 中的迁移功能能够以递增方式更新数据库架构，使其与应用程序的数据模型保持同步，同时保留数据库中的现有数据。</span><span class="sxs-lookup"><span data-stu-id="2b078-105">The migrations feature in EF Core provides a way to incrementally update the database schema to keep it in sync with the application's data model while preserving existing data in the database.</span></span>

<span data-ttu-id="2b078-106">简要地说，迁移的方式如下：</span><span class="sxs-lookup"><span data-stu-id="2b078-106">At a high level, migrations function in the following way:</span></span>

* <span data-ttu-id="2b078-107">当引入数据模型更改时，开发人员使用 EF Core 工具添加相应的迁移，以描述使数据库架构保持同步所需的更新。EF Core 将当前模型与旧模型的快照进行比较，以确定差异，并生成迁移源文件；文件可在项目的源代码管理中进行跟踪，如任何其他源文件。</span><span class="sxs-lookup"><span data-stu-id="2b078-107">When a data model change is introduced, the developer uses EF Core tools to add a corresponding migration describing the updates necessary to keep the database schema in sync. EF Core compares the current model against a snapshot of the old model to determine the differences, and generates migration source files; the files can be tracked in your project's source control like any other source file.</span></span>
* <span data-ttu-id="2b078-108">生成新的迁移后，可通过多种方式将其应用于数据库。</span><span class="sxs-lookup"><span data-stu-id="2b078-108">Once a new migration has been generated, it can be applied to a database in various ways.</span></span> <span data-ttu-id="2b078-109">EF Core 在一个特殊的历史记录表中记录所有应用的迁移，使其知道哪些迁移已应用，哪些迁移尚未应用。</span><span class="sxs-lookup"><span data-stu-id="2b078-109">EF Core records all applied migrations in a special history table, allowing it to know which migrations have been applied and which haven't.</span></span>

<span data-ttu-id="2b078-110">本页的其余部分是使用迁移的分步式入门指南。</span><span class="sxs-lookup"><span data-stu-id="2b078-110">The rest of this page is a step-by-step beginner's guide for using migrations.</span></span> <span data-ttu-id="2b078-111">有关更深入的信息，请参阅本节中的其他页面。</span><span class="sxs-lookup"><span data-stu-id="2b078-111">Consult the other pages in this section for more in-depth information.</span></span>
